Desarrollo de MeetSure

El equipo ha desarrollado la aplicación en 3 sprints

El objetivo que se ha propuesto para la primera iteración era tener desarrollada una aplicación básica en la que se puedan crear y gestionar usuarios, y publicar y gestionar actividades de distintas formas. Esta iteración no iba generar un producto entregable hacia el cliente pero si un producto interno que podriamos empezar a usar para realizar pruebas de carga y de usabilidad de la base de datos en MySQL, y la aplicación Android.

El objetivo para la segunda iteración era expandir las funcionalidades que ofrece la aplicación básica, dándole un valor añadido. Esta iteración, al igual que en la primera iteración, tampoco generó un producto entregable hacia el cliente, pero tras añadir estas funcionalidades estaremos más cerca del producto final.

Para la correcta organización del equipo y el desarrollo de la aplicación hemos utilizado las siguientes técnicas:

Hemos utilizado tambien un Excel para llevar la evolución del producto de manera ordenada y correcta:

  • Burndown

    Representación gráfica del trabajo por hacer del proyecto en el tiempo. El trabajo remanente (o backlog) se muestra en el eje vertical y el tiempo en el eje horizontal.

  • Historias de Usuario

    Representación de un requisito escrito en una o dos frases utilizando el lenguaje común del usuario.

  • Tabla de Planificación

    Tablas donde se muestran las tareas realizadas y el tiempo neto y real que se ha gastado para realizarlas.

  • Colaboratividad

    Gráficas donde se muestra el tiempo que ha gastado un miembro del equipo para realizar sus tareas.
